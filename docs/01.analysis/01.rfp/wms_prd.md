# WMS 실습 프로젝트 PRD (교육용)

## 1. 목적

Cursor 에이전트 활용한 개발 실습용 WMS(창고관리시스템).
멀티 에이전트 협업으로 처음부터 끝까지 자동 완성되는 과정을 체험한다.

## 2. 기술 스택

- 백엔드: Spring Boot 3.x, Java 17, MyBatis, PostgreSQL
- 프론트: Vue 3 (CDN), Element Plus, Tailwind CSS, Axios
- 빌드: Gradle
- 패키지: `com.execnt.wms` (그룹: `com.carfix`)
- DB: 스키마 `wms`, 테이블 접두사 `twms_`

## 3. 제외

- 인증/로그인/JWT 일체
- 외부 시스템 연계
- 배치 스케줄러 (수동 API로 대체)
- 마스터 CRUD 화면 (초기 데이터 INSERT로 해결)

## 4. 기능 요구사항

### F1. 입고 관리

외부에서 수신된 입고 주문을 관리하는 화면을 제공한다.
사용자는 입고 주문을 조회하고, 검수 → 입고 확정까지 단계별로 처리할 수 있어야 한다.

- 주문 목록을 센터별, 상태별로 필터링하여 조회할 수 있다
- 주문 상세에서 품목별 수량을 확인할 수 있다
- 상태를 단계별로 진행할 수 있다 (검수시작 → 검수완료 → 입고확정)
- 허용되지 않은 상태 변경은 거부되고 사유가 표시된다

### F2. 출고 관리

외부에서 수신된 출고 주문을 관리하는 화면을 제공한다.
사용자는 출고 주문을 조회하고, 피킹 → 출고 확정까지 단계별로 처리할 수 있어야 한다.

- F1과 동일한 UX 패턴을 따른다
- 상태를 단계별로 진행할 수 있다 (피킹시작 → 피킹완료 → 출고확정)

### F3. 상태 현황 대시보드

입출고 처리 현황을 한눈에 파악할 수 있는 대시보드를 제공한다.

- 입고/출고 상태별 건수를 요약하여 보여준다
- 최근 상태 변경 이력을 확인할 수 있다
- 특정 주문의 전체 상태 변경 이력을 추적할 수 있다

### F4. 상태 전이 규칙

모든 상태 변경은 서버에서 사전 정의된 전이 규칙에 따라 검증된다.

- 허용된 상태 전이만 가능하다
- 허용되지 않은 전이 시도 시 명확한 에러 메시지를 반환한다
- 상태 변경 시 이력이 자동으로 기록된다
- 상태 변경과 이력 기록은 같은 트랜잭션 내에서 처리된다

## 5. 비기능 요구사항

| 구분 | 요구 |
|------|------|
| 사용성 | 상태 변경 시 확인/피드백 제공. 입력 검증과 안내 문구 |
| 데이터 | API 응답은 Map<String, Object> 중심. SQL은 MyBatis XML에만 작성 |
| 가시성 | 주요 상태 변경에 대한 이력 추적 가능 |

## 6. 업무 도메인 전제

- 상품 단위로 관리한다
- 물류센터(창고) 단위로 운영한다
- 입출고 상태는 공통 코드로 관리한다
- 상태 전이 흐름:
  - 입고: 주문생성 → 검수중 → 검수완료 → 입고확정
  - 출고: 주문생성 → 피킹중 → 피킹완료 → 출고확정
