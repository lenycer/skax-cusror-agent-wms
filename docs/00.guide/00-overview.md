# 00. 개요 — 전체 실습 로드맵

## 1. 이 실습에서 만드는 것

이 가이드를 끝까지 따라가면 두 가지를 얻습니다.

첫째, Cursor IDE의 다섯 가지 하네스 요소를 실제 프로젝트 저장소 안에 파일로 구성하는 방법을 익힙니다. 둘째, 그 하네스를 활용해 WMS(창고관리시스템) 도메인의 문서 생성부터 코드 개발까지를 에이전트 기반으로 수행하는 워크플로를 체험합니다.

## 2. WMS 도메인 요약

이 저장소는 물류센터(창고) 단위로 상품·재고를 관리하는 WMS를 다룹니다. 입고 주문과 출고 주문은 외부 연계를 전제하며, 검수→입고 확정, 피킹→출고 확정 같은 표준 흐름을 따릅니다. 상태·단계는 공통 코드로 관리합니다.

제품 요구사항의 전문은 `docs/01.analysis/01.rfp/wms_prd.md`에 있고, 기술 스택과 DB 규약은 루트 README와 `.cursor/rules/`의 프로젝트 지침에 정리되어 있습니다.

## 3. Cursor 하네스 — 다섯 가지 요소

Cursor에서 에이전트의 동작을 제어하는 설정은 모두 프로젝트 루트의 `.cursor/` 폴더에 파일로 저장됩니다. 다섯 가지 요소가 각각 다른 역할을 합니다.

**Rules** (`.cursor/rules/*.mdc`)는 에이전트가 코드를 작성하거나 대화할 때 항상 참고해야 하는 지침입니다. "프로젝트 헌법"에 해당합니다. 파일을 열거나 편집할 때 경로 패턴(globs)에 따라 자동으로 적용되거나, 항상 적용되도록 설정할 수 있습니다.

**Commands** (`.cursor/commands/*.md`)는 채팅 입력창에서 `/이름`으로 호출하는 표준 작업 절차문입니다. 팀원 누구나 같은 `/gen-req`를 입력하면 동일한 절차와 산출물 경로로 작업이 시작됩니다.

**Skills** (`.cursor/skills/**/SKILL.md`)는 특정 작업 유형에 대한 참고서입니다. 예를 들어 "Task 정의서는 이 8개 섹션으로 작성한다"거나 "DDL은 이 네이밍 패턴을 따른다"는 것을 에이전트가 필요할 때 참조합니다.

**Agents** (`.cursor/agents/*.md`)는 역할이 정해진 보조 에이전트 정의입니다. 백엔드 전담, 프론트 전담, DB 전담 등으로 나눠 채팅에서 이름으로 호출합니다. Cursor 3.0의 병렬 에이전트 실행과 결합하면 여러 역할이 동시에 작업할 수 있습니다.

**Hooks** (`.cursor/hooks.json` + `.cursor/hooks/*.sh`)는 에이전트가 파일을 읽거나, 수정하거나, 셸 명령을 실행하기 직전·직후에 자동으로 실행되는 스크립트입니다. 위험한 명령을 차단하거나, 편집 후 자동 포맷을 걸거나, 민감 파일 접근을 막는 데 사용합니다.

이 다섯 요소의 관계를 한 문장으로 정리하면 이렇습니다: **Rules가 맥락을 제공하고, Commands가 진입점을 열고, Skills가 형식을 잡고, Agents가 역할을 나누고, Hooks가 실행 시점에 개입합니다.**

## 4. 실습 순서


| 단계  | 문서                        | 하는 일                                           |
| --- | ------------------------- | ---------------------------------------------- |
| 1   | `01-repository-layout.md` | 저장소 폴더 구조와 문서 경로 규약 확인                         |
| 2   | `02-rules.md`             | Rules 개념 이해 + 프로젝트용 `.mdc` 파일 구축               |
| 3   | `03-commands.md`          | Commands 개념 이해 + 슬래시 커맨드 7개 구축                 |
| 4   | `04-skills.md`            | Skills 개념 이해 + [SKILL.md](http://skill.md/) 구축 |
| 5   | `05-agents.md`            | Agents 개념 이해 + 서브에이전트 정의 구축                    |
| 6   | `06-hooks.md`             | Hooks 개념 이해 + 설정 + 실제 동작 확인                    |
| 7   | `07-verification.md`      | 전체 형식 점검 + 라우팅 표 + 스모크 테스트                     |
| 8   | `08-sequential-dev.md`    | PRD 범위 고정 + 에이전트 개별 순차 실행으로 WMS 개발             |
| 9   | `09-orchestration.md`     | 소스 초기화 후 오케스트레이터에게 전체 위임·자동 완성                 |


01~06은 `.cursor/` 하위에 설정 파일을 만드는 구축 단계이고, 07에서 전체를 점검합니다. 08에서 각 에이전트를 하나씩 호출해 단계별로 개발하고, 09에서 오케스트레이터 에이전트 하나에게 전체를 맡겨 자동으로 완성하는 흐름을 체험합니다.

## 5. 진행 방식

각 문서는 동일한 구조를 따릅니다: "이 요소가 무엇인지"를 먼저 설명하고, “어떤 파일을 만드는지” 표로 보여준 뒤, "Agent에게 줄 프롬프트 예시"를 제공합니다.

프롬프트 예시를 Cursor 채팅에 복사해 실행하면 에이전트가 파일을 생성하고, 직접 파일을 열어 편집해도 됩니다. 두 방식을 섞어도 무방합니다.
